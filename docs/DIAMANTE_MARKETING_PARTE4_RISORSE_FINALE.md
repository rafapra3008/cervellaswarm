## 5. RISPOSTA TEMPLATE - SITUAZIONI COMUNI

### 5.1 Commento Positivo

**Template:**
```
Thank you! Really appreciate the feedback.

[If they mention specific feature]:
That's exactly why we built [feature] - glad it resonates!

[If they ask question]:
[Answer] - and we're documenting this in the FAQ now.

Would love to hear how you use it if you try it out!
```

### 5.2 Critica Costruttiva

**Template:**
```
Great point, and you're right about [acknowledge concern].

[Explain reasoning OR commit to improvement]:
- If valid concern: "This is on our roadmap - [timeline/approach]"
- If design decision: "We chose [X] because [reason], but [alternative] is possible"

Thanks for taking time to give feedback - this helps us improve!
```

### 5.3 Commento Negativo/Trolling

**Template:**
```
[If valid criticism]:
Appreciate the honest feedback. [Address specific point].

[If trolling/unconstructive]:
[Don't engage OR simple]:
Thanks for checking it out. Open to specific feedback if you have suggestions!
```

### 5.4 Feature Request

**Template:**
```
Love this idea!

[Clarifying questions if needed]

Mind opening a GitHub issue so we can track it and discuss implementation?
Link: [repo]/issues

[If popular request]:
This is actually the #3 most requested feature so far - definitely on our radar!
```

### 5.5 Bug Report

**Template:**
```
Thanks for reporting! This is definitely a bug.

I'm looking into it now. Could you share:
- OS/environment: [?]
- Steps to reproduce: [?]
- Expected vs actual behavior: [?]

GitHub issue here to track: [link]

ETA for fix: [timeframe]
```
