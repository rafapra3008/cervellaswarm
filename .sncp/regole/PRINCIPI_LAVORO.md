# PRINCIPI LAVORO - CervellaSwarm

> **NON RIPETERE!** Queste regole sono SCRITTE. Leggile e applicale.

---

## 1. Lavoro Grande = Lavori Piccoli

```
NESSUN LAVORO E' GRANDE SE LO DIVIDI IN LAVORI PICCOLI.

Task > 30 min? DIVIDILO.
Task complesso? DIVIDILO.
Task che fa paura? DIVIDILO.

Ogni pezzo: max 30 min, verificabile, completabile.
```

---

## 2. Sempre Tempo, Mai Fretta

```
ABBIAMO TEMPO! SEMPRE!

Non correre.
Non saltare passi.
Non "dopo lo sistemo".

Fatto BENE > Fatto VELOCE.
```

---

## 3. Organizzare PRIMA, Eseguire DOPO

```
STOP → DIVIDI → SCRIVI → ESEGUI → VERIFICA

1. STOP - Non iniziare subito
2. DIVIDI - In pezzi piccoli
3. SCRIVI - Lista todo o roadmap
4. ESEGUI - Uno alla volta
5. VERIFICA - Dopo ogni pezzo
```

---

## 4. Una Cosa Alla Volta

```
UNA COSA ALLA VOLTA, FATTA BENE.

Non parallelizzare a caso.
Non saltare da un task all'altro.
Finisci. Verifica. Prossimo.
```

---

## 5. Workflow Sessione

```
[INIZIO 15 min]
├─ Leggi PROMPT_RIPRESA + NORD
├─ Obiettivo chiaro con Rafa
├─ Plan mode (breakdown)
└─ Valida piano insieme

[ESECUZIONE 60 min]
├─ Regina autonoma
├─ Spawna worker se serve
├─ Report solo milestone/blocco
└─ Rafa hands-off

[CHIUSURA 15 min]
├─ Review + test
├─ Deploy se pronto
├─ Checkpoint (NORD, PROMPT_RIPRESA, git)
└─ Handoff per prossima sessione
```

---

## 6. Chi Decide Cosa

| Chi | Cosa |
|-----|------|
| **RAFA** | Strategia, priorita, scope, business |
| **REGINA** | Breakdown, implementazione, tool, worker |
| **INSIEME** | Piano, architettura, deploy |

---

## 7. Comunicazione

| Quando | Dettaglio |
|--------|-----------|
| INIZIO | Alto (piano) |
| DURANTE | Basso (solo blocchi) |
| FINE | Medio (risultato + next) |

**NO:** Progress continuo, dettagli tecnici non richiesti
**SI:** Risultati, blocchi, decisioni strategiche
