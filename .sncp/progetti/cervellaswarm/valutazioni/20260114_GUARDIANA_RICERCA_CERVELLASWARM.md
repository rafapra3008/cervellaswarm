## 3. SNCP - QUANTO E' UNICO?

### Confronto con Soluzioni di Memoria AI

| Sistema | Tipo | Pro | Contro |
|---------|------|-----|--------|
| **SNCP (nostro)** | File-based strutturato | Semplice, persistente, versionabile | Manuale |
| **Mem0** | API cloud | Scalabile, produzione-ready | Lock-in, costo |
| **A-MEM** | Agentic Zettelkasten | Dinamico, interconnesso | Complessita |
| **Claude CLAUDE.md** | File singolo | Nativo | Limitato |

### Ricerca Accademica Recente

**Paper rilevanti (2025-2026):**
- "Memory in the Age of AI Agents" (arXiv 2512.13564)
- "A-MEM: Agentic Memory for LLM Agents" (arXiv 2502.12110)
- "Intrinsic Memory Agents" (arXiv 2508.08997)

**Key insight:**
> "65% of enterprise AI failures in 2025 were attributed to context drift or memory loss during multi-step reasoning."

**Fonte:** [Calibraint LLM Development 2026](https://www.calibraint.com/blog/llm-development-services-in-2026)

### Verdetto SNCP

```
SNCP NON e' completamente unico come CONCETTO
(altri stanno lavorando su memoria strutturata)

SNCP E' unico come IMPLEMENTAZIONE:
- File-based (versionabile con git)
- Struttura progetto (progetti/decisioni/idee)
- Integrato con workflow multi-agent
- SEMPLICE (non richiede infra esterna)
```
