### L'Utente NON Deve Mai Pensare:

```

❌ "Ce la faro?"
❌ "Devo lavorare X ore al giorno"

```

## COME QUESTO INFLUENZA IL WIZARD

### Session Resume

```
L'utente riapre dopo 1 settimana.

SBAGLIATO:
"Sono passati 7 giorni! Sei in ritardo!"

CORRETTO:
"Bentornato! Ecco dove eravamo.
Ecco cosa abbiamo fatto.
Ecco il prossimo passo.
Facciamolo insieme?"
```

### Progress Display

```
SBAGLIATO:
"Progresso: 35% - Tempo stimato: 3 settimane"

CORRETTO:
"Progresso: 35%
Prossimo passo: Implementare autenticazione
Tempo per questo passo: ~1 sessione"
```

### Messaggi

```
SBAGLIATO:
"Hai lavorato solo 2 ore questa settimana"

CORRETTO:
"Hai fatto 3 progressi questa settimana!
Ogni passo conta."
```

## I PRINCIPI DA IMPLEMENTARE

### 1. MAI mostrare timeline totali

```
NO: "Questo progetto richiede 3 mesi"
SI: "Prossimo passo: X"
```

### 2. MAI giudicare frequenza uso

```
NO: "Non ti vedo da 2 settimane..."
SI: "Bentornato! Riprendiamo da dove eravamo."
```

### 3. SEMPRE focus sul prossimo passo

```
Ogni sessione = 1 passo chiaro
Non importa quanto grande e il progetto
Il passo di oggi e quello che conta
```

### 4. CELEBRARE ogni progresso

```
"Passo completato!"
"Sei avanzato!"
"Ottimo lavoro oggi!"

Non: "Solo X% fatto, manca ancora tanto"
```

## COME IMPLEMENTARE

### Nel Wizard Init

```
Domanda: "Quanto vuoi lavorare per sessione?"
Opzioni:
- Sessioni brevi (30-60 min)
- Sessioni medie (1-2 ore)
- Sessioni lunghe (2+ ore)
- Come viene

NON chiedere: "Quando vuoi finire?"
```

### Nel Resume

```
Mostrare:
- Dove eravamo
- Cosa abbiamo fatto (celebrare!)
- Prossimo passo suggerito

NON mostrare:
- "Tempo dall'ultimo accesso"
- "Sei in ritardo rispetto a..."
- Deadline o timeline
```

### Nei Messaggi

```
Tono:
- Accogliente
- Senza giudizio
- Focus sul presente
- Celebrativo

NON:
- Pressione
- Urgenza
- Confronto con altri
- Timeline
```
