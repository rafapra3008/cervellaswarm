## 1. ANALISI VALORE PERCEPITO

### 1.1 Situazione PRIMA del software

**Processo manuale contabilità:**
- Upload PDF Ericsoft manuale
- Inserimento dati in Excel riga per riga
- Ricerca duplicati manuale
- Controllo totali manuale
- Preparazione file Spring manuale
- Gestione stagioni separata

**Tempo stimato:** 10-15 ore/mese = 120-180 ore/anno

### 1.2 Situazione DOPO il software

**Processo automatizzato:**
- Upload PDF con parsing automatico
- Deduplicazione automatica
- Validazione totali in tempo reale
- Generazione Excel automatica
- Export Spring predisposto
- Filtri dinamici per stagione

**Tempo attuale:** 2-3 ore/mese = 24-36 ore/anno

**Tempo risparmiato:** 96-144 ore/anno

### 1.3 Calcolo Valore Monetario

```
Scenario conservativo:
- Tempo risparmiato: 100 ore/anno
- Valore orario contabile: 150 EUR/h
- VALORE ANNUALE: 15.000 EUR

Scenario realistico:
- Tempo risparmiato: 120 ore/anno
- Valore orario: 150 EUR/h
- VALORE ANNUALE: 18.000 EUR
```

**ROI regola 10x:** Per 18.000 EUR valore annuale → prezzo max 1.800 EUR/anno

### 1.4 Benefici Qualitativi (non monetizzabili ma reali)

- **Zero errori** - Validazione automatica impedisce errori
- **Audit trail completo** - Ogni modifica tracciata
- **Accessibilità cloud** - Lavoro da ovunque
- **Serenità** - "Funziona sempre, non devo pensarci"
- **Professionalità** - Sistema proprietario, non Excel

## 3. STRATEGIA PRICING

### 3.1 Modello Raccomandato: IBRIDO

**Licenza Perpetua + Manutenzione Mensile**

Perché questo modello:
- Cliente già usa il software (cambio doloroso)
- Soluzione custom = alto switching cost
- Famiglia vuole certezza proprietà
- Manutenzione garantisce aggiornamenti

### 3.2 Pricing Suggerito

```
OPZIONE A (Conservativa):
- Licenza perpetua: 5.000 EUR (una tantum)
- Manutenzione: 150 EUR/mese (1.800 EUR/anno)
- TOTALE anno 1: 6.800 EUR
- TOTALE successivi: 1.800 EUR/anno

OPZIONE B (Valore reale):
- Licenza perpetua: 8.000 EUR (una tantum)
- Manutenzione: 200 EUR/mese (2.400 EUR/anno)
- TOTALE anno 1: 10.400 EUR
- TOTALE successivi: 2.400 EUR/anno

OPZIONE C (Premium):
- Licenza perpetua: 12.000 EUR (una tantum)
- Manutenzione: 250 EUR/mese (3.000 EUR/anno)
- TOTALE anno 1: 15.000 EUR
- TOTALE successivi: 3.000 EUR/anno
```

### 3.3 Cosa Include Ogni Componente

**Licenza Perpetua (one-time):**
- Diritto d'uso illimitato software attuale
- Codice sorgente in proprietà
- Database e infrastruttura
- Documentazione completa
- Training iniziale

**Manutenzione Mensile (ricorrente):**
- Supporto tecnico prioritario
- Bug fix entro 48h
- Aggiornamenti sicurezza
- Backup automatici verificati
- Monitoraggio server
- Piccoli miglioramenti funzionali

**NON include (sviluppo extra):**
- Feature completamente nuove
- Integrazioni con altri sistemi
- Refactoring maggiori

### 3.4 ROI per il Cliente

## 4. PSICOLOGIA DEL PREZZO

### 4.1 Ancoraggio (Anchoring)

```
SBAGLIATO:
"Il software costa 8.000 EUR"

CORRETTO:
"Senza questo software, gestione contabilità richiede 120 ore/anno.
Al valore orario di un contabile (150 EUR/h), sono 18.000 EUR/anno.
Questo software automatizza tutto e costa 8.000 EUR una tantum.
Si ripaga in 5 mesi."
```

**Principio:** Presenta PRIMA il valore (18.000 EUR), POI il prezzo (8.000 EUR)

### 4.2 Framing (Inquadratura)

```
SBAGLIATO:
"Licenza 8.000 + manutenzione 200/mese"

CORRETTO:
"Proprietà completa del software a 8.000 EUR.
Include supporto continuo a solo 7 EUR/giorno."
```

**200 EUR/mese = 7 EUR/giorno** (psicologicamente più basso)

### 4.3 Loss Aversion (Avversione alla Perdita)

```
MESSAGGIO:
"Senza questo sistema, ogni mese perdete 8-10 ore in lavoro manuale.
In un anno, sono 2-3 settimane di lavoro perse.
Questo investimento le recupera per sempre."
```

**Principio:** Enfatizza cosa PERDONO senza il software, non solo cosa guadagnano

### 4.4 Sunk Cost (Costo Sommerso)

```
REALTA:
- Software sviluppato in 6+ mesi
- Personalizzazioni specifiche Ericsoft
- Database storico già caricato
- Workflow rodato e collaudato

MESSAGGIO:
"Sistema già attivo e testato, con 2+ anni di dati storici.
Ripartire da zero con altro software richiederebbe mesi."
```

## 6. COME PRESENTARE L'OFFERTA

### 6.1 Timing & Setting

**QUANDO:**
- Dopo aver verificato che sistema funziona bene
- Durante review contabilità (quando vedono valore)
- NON durante periodo stressante

**COME:**
- Meeting dedicato (non email fredda)
- Presentazione visuale (non solo numero)
- Preparare demo prima/dopo (mostrare valore)


**OBIEZIONE 1: "È caro!"**

Risposta:
"Capisco. Confrontiamo: software generico come Sage costa
600 EUR/anno ma NON fa parsing Ericsoft, NON gestisce
stagioni lodge, NON export Spring. Richiedereste comunque
10 ore/mese di lavoro manuale = 18.000 EUR/anno valore tempo.

Questo sistema costa 10.400 EUR anno 1, vi fa risparmiare
18.000 EUR/anno. È un GUADAGNO di 7.600 EUR anno 1!"

---

**OBIEZIONE 2: "E se Rafa non è più disponibile?"**

Risposta:
"Ottima domanda. Per questo propongo:
1. Codice sorgente in vostra proprietà (nella licenza)
2. Documentazione completa per sviluppatore futuro
3. Codice in escrow (deposito sicuro)
4. Training a personale vostro se volete

Il software È VOSTRO, non dipendete da me."

---

**OBIEZIONE 3: "Possiamo pagare meno?"**

Risposta:
"Posso offrire sconto 6% se pagate anno intero in anticipo:
9.800 EUR invece di 10.400 EUR.

O possiamo dilazionare: 4.000 EUR ora, 4.000 EUR tra 3 mesi,

---

Confronto: con SaaS subscription, appena smetti di pagare
perdi tutto. Qui POSSEDETE il software."
