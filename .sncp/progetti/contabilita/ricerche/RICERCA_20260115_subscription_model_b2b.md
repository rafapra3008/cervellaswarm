## 1. TREND 2026: Modelli Subscription B2B

### 1.1 Panorama Generale

Il mercato B2B SaaS nel 2026 è caratterizzato da:

- **67% delle aziende** usa modelli ibridi (combinazione di più approcci)
- **Declino flat-rate puro:** Solo 12-18% raccomanda pricing flat oltre $5M ARR
- **Flessibilità crescente:** Modelli devono adattarsi a usage, segmenti, custom deals
- **AI impact:** Pricing per-feature sta crescendo per monetizzare capacità AI

**Fonte principale:** [Software Monetization Models 2026](https://www.getmonetizely.com/articles/software-monetization-models-and-strategies-for-2026-the-complete-guide)

### 1.2 Modelli Principali

| Modello | Descrizione | Pro | Contro |
|---------|-------------|-----|--------|
| **Flat-rate subscription** | Canone fisso mensile/annuale | Semplice, prevedibile | Non scala con valore |
| **Per-user** | Prezzo per utente attivo | Scala con team size | Limita adoption |
| **Usage-based** | Paga per transazioni/volume | Fair, scala con uso | Revenue imprevedibile |
| **Tiered** | Pacchetti Small/Medium/Large | Upsell chiaro | Rigido |
| **Hybrid** | Combinazione dei sopra | Flessibile, ottimizza revenue | Più complesso |

**Per software custom PMI:** Il modello **Flat-rate + minimo contrattuale** è il più adatto perché:
- Cliente piccolo preferisce prevedibilità
- Non ci sono "utenti multipli" da monetizzare
- Il valore è nel sistema completo, non nell'usage

## 3. COSA INCLUDERE NEL CANONE MENSILE

### 3.1 Componenti Standard

Basandosi su analisi mercato e best practices, il canone mensile dovrebbe includere:

#### ✅ Licenza Software
- Diritto d'uso del software custom
- Aggiornamenti versione inclusi
- Nessun limite utenti (per PMI piccola)

#### ✅ Hosting & Infrastruttura
- Server hosting (cloud o dedicato)
- Database hosting e backup
- SSL certificate
- Storage dati
- Banda mensile

#### ✅ Manutenzione
- Bug fixing
- Security patches
- Aggiornamenti minori
- Performance optimization

#### ✅ Supporto
- Email/ticket support (orario lavorativo)
- Tempo risposta garantito (es: 24-48h)
- Remote assistance quando necessario

### 3.2 Servizi Opzionali (Extra)

Servizi NON inclusi nel canone base:
- ❌ Sviluppo nuove funzionalità custom
- ❌ Formazione on-site
- ❌ Consulenza strategica/organizzativa
- ❌ Integrazione con sistemi terzi
- ❌ Supporto prioritario h24/7

**Fonte:** [Website Maintenance Packages](https://www.webyking.com/packages/website-maintenance-packages/), [Monthly Webmaster Plans](https://barketing.co/monthly-webmaster-plans/)

### 3.3 Modello "Support Hours"

Alcune soluzioni includono ore di supporto mensili:
- **Basic:** 30 min/mese support incluso
- **Professional:** 60-90 min/mese support incluso
- **Extra hours:** Fatturate a parte (€80-150/ora)

Questo approccio è comune nel mercato e aiuta a:
- Bilanciare richieste cliente
- Evitare abusi di supporto "illimitato"
- Revenue prevedibile per vendor

## 4. DURATA CONTRATTO: Minimo Contrattuale

### 4.1 Standard di Mercato

| Durata | % Adozione | Pro | Contro |
|--------|------------|-----|--------|
| **Rolling (month-to-month)** | ~15% | Flessibilità cliente | Alto churn, no garanzia revenue |
| **6 mesi minimo** | ~20% | Valutazione iniziale | Troppo breve per ROI vendor |
| **12 mesi minimo** | ~50% | Bilanciato, standard | Impegno moderato |
| **24 mesi minimo** | ~15% | Revenue garantito | Barriera entry alta |

**Fonte:** [Contract Duration Best Practices](https://www.hyperstart.com/blog/contract-duration/)

### 4.2 Raccomandazione per Software Custom

**MINIMO 12 MESI è lo standard raccomandato** perché:

✅ **Per il Vendor:**
- Copre costi setup iniziale (deployment, formazione)
- Tempo sufficiente per onboarding e supporto iniziale
- Garantisce ROI su investimento relazione cliente

✅ **Per il Cliente:**
- Prezzo fisso per 12 mesi (protezione da aumenti)
- Tempo adeguato per adottare il software
- Non troppo vincolante (vs 24-36 mesi)

**Clausola tipo:**
```
"Durata iniziale: 12 mesi dalla data di attivazione.
Dopo il periodo iniziale, rinnovo automatico mensile con preavviso 60 giorni."
```

### 4.3 Struttura Temporale Suggerita

```
MESI 1-3:   Setup, formazione, supporto intensivo
MESI 4-9:   Uso normale, supporto standard
MESI 10-12: Valutazione, decisione rinnovo
MESE 13+:   Rolling mensile o nuovo contratto annuale
```

**Nota:** Per software complesso con change management, alcuni vendor richiedono 24-36 mesi per minimizzare il rischio di abbandono prima dell'adozione completa.

**Fonte:** [SaaS Subscription Guide](https://www.sastrify.com/blog/monthly-annual-saas-subscriptions)

## 5. CLAUSOLE TIPICHE

### 5.1 Aumento Annuale (Price Escalation)

**Standard di mercato:**

| Range | Casistica |
|-------|-----------|
| **3% annuo** | Conservativo, USA standard |
| **3-5% annuo** | Più comune, legato a inflazione/CPI |
| **5-7% annuo** | Enterprise, grandi vendor (Salesforce, SAP) |
| **7-10% annuo** | Raro, solo con clausola cap |

**Fonte:** [SaaS Price Increase Strategies](https://sumeruequity.com/price-increases/)

**Raccomandazione:**
```
"Il canone mensile potrà essere adeguato annualmente in misura non superiore
all'indice ISTAT dei prezzi al consumo, con un massimo del 5% annuo."
```

**Perché questa clausola:**
- Protegge vendor da inflazione costi (server, energia, etc)
- Incentiva cliente a firmare pluriennale (prezzo bloccato)
- Equa per entrambe le parti
- Cap del 5% evita shock pricing

### 5.2 Cancellazione (Termination)

**Standard di mercato:**
- **Preavviso:** 30-90 giorni (60 giorni è il più comune)
- **Durante minimo contrattuale:** Generalmente NON consentita, o con penale
- **Dopo minimo contrattuale:** Libera con preavviso

**Fonte:** [Software Subscription Termination Terms](https://aaronhall.com/software-subscription-termination-terms/)

**Raccomandazione:**
```
"Durante i primi 12 mesi: nessuna cancellazione consentita salvo cause gravi.
Dopo 12 mesi: cancellazione con preavviso scritto di 60 giorni."
```

**Opzione con penale:**
```
"Il Cliente può recedere anticipatamente pagando il 50% dei canoni residui
fino alla scadenza del periodo minimo."
```

### 5.3 Rinnovo Automatico

**Best practice:** Notifica automatica prima del rinnovo

**Requisiti legali (es: California, Colorado):**
- Per contratti ≥12 mesi: notifica 15-45 giorni prima del rinnovo
- Per alcuni stati USA: 25-45 giorni (Colorado), 30-60 giorni (Delaware)

**Fonte:** [California Auto-Renewal Law](https://prosperstack.com/blog/california-automatic-renewal-law/)

**Raccomandazione (Italia):**
```
"Il contratto si rinnova automaticamente di mese in mese dopo il periodo iniziale.
Il fornitore invierà reminder 30 giorni prima della scadenza del periodo minimo."
```

### 5.4 Service Level Agreement (SLA)

Per software custom in subscription, tipicamente include:

- **Uptime:** 99% mensile (equivale a ~7 ore downtime/mese accettabili)
- **Tempo risposta supporto:** 24-48h per ticket non critici
- **Backup:** Giornaliero, retention 30 giorni
- **Security patches:** Applicati entro 7 giorni dalla disponibilità

**Nota:** SLA enterprise (99.9%, 99.99%) sono overkill per PMI e aumentano costi significativamente.

## 6. PROPRIETÀ INTELLETTUALE nel Modello Subscription

### 6.1 Differenza Chiave: Custom vs SaaS

| Aspetto | Software Custom | SaaS Standard |
|---------|-----------------|---------------|
| **Sviluppo** | Fatto su misura per cliente | Pre-esistente, multi-tenant |
| **Ownership** | Questione da definire contrattualmente | Developer mantiene sempre |
| **Accesso** | Può includere source code | Mai accesso a source code |

**Fonte:** [IP Ownership Custom Software](https://aaronhall.com/ip-ownership-custom-software-development-projects/)

### 6.2 Modelli di Ownership

#### Opzione A: CLIENTE POSSIEDE IL SOFTWARE

```
Il cliente possiede tutti i diritti IP sul software custom sviluppato.
Il vendor fornisce:
- Licenza d'uso durante la subscription
- Hosting e manutenzione
- Accesso source code alla scadenza (o su richiesta)

Vantaggi cliente:
✅ Protezione investimento
✅ Può cambiare vendor in futuro
✅ Può rivendere (se business cambia)

Svantaggi vendor:
❌ Non può riusare codice per altri clienti
❌ Meno controllo su modifiche future
```

#### Opzione B: VENDOR POSSIEDE, CLIENTE HA LICENZA

```
Il vendor mantiene ownership del software.
Il cliente riceve:
- Licenza d'uso esclusiva (se custom puro)
- Diritto d'accesso finché paga subscription
- NESSUN accesso source code

Vantaggi vendor:
✅ Controllo totale su IP
✅ Può riutilizzare componenti per altri progetti
✅ Leverage per mantenere cliente

Svantaggi cliente:
❌ Dipendenza totale dal vendor
❌ Perde accesso se cancella subscription
❌ Non può modificare autonomamente
```

**Fonte:** [Software Ownership Models](https://senlainc.com/blog/software-ownership-who-owns-the-software/)

### 6.3 Modello IBRIDO (Raccomandato)

```
DURANTE SUBSCRIPTION:
- Vendor mantiene IP
- Cliente ha licenza d'uso esclusiva
- Vendor fornisce hosting + manutenzione

ALLA FINE SUBSCRIPTION (se cliente cancella):
OPZIONE 1: Cliente può acquistare IP (one-time fee)
OPZIONE 2: Cliente riceve source code + licenza perpetua (exit fee)
OPZIONE 3: Vendor riprende software, cliente esporta dati

ESCROW AGREEMENT:
- Source code depositato presso terza parte
- Rilasciato a cliente se vendor fallisce/abbandona
```

**Perché è bilanciato:**
- ✅ Vendor protetto durante subscription attiva
- ✅ Cliente non "ostaggio" (ha exit option)
- ✅ Incentiva entrambe le parti a mantenere rapporto
- ✅ Escrow protegge cliente da fallimento vendor

**Fonte:** [IP Ownership in Custom Software Deals](https://aaronhall.com/ip-ownership-in-custom-software-development-deals/)

## 8. CONFRONTO: Subscription vs Licenza Perpetua

### 8.1 Pro e Contro dei Due Modelli

| Aspetto | SUBSCRIPTION | PERPETUAL LICENSE |
|---------|-------------|-------------------|
| **Costo iniziale** | ✅ Basso (primo mese) | ❌ Alto (una tantum) |
| **Costo totale 5 anni** | ❌ Più alto nel lungo periodo | ✅ Più basso se ammortizzato |
| **Prevedibilità costi** | ✅ Canone fisso mensile | ⚠️ Unpredictable (maintenance) |
| **Revenue vendor** | ✅ Ricorrente, prevedibile | ❌ Una tantum, imprevedibile |
| **Barriera ingresso cliente** | ✅ Bassa | ❌ Alta |
| **Aggiornamenti** | ✅ Sempre inclusi | ⚠️ Solo se in maintenance |
| **Churn risk** | ❌ Cliente può andarsene | ✅ Ha già pagato |
| **Supporto** | ✅ Incluso nel canone | ⚠️ Extra (maintenance) |
| **Hosting** | ✅ Incluso | ❌ Cliente si arrangia |
| **Ownership** | ⚠️ Dipende da contratto | ✅ Cliente spesso possiede |

**Fonte:** [Perpetual vs Subscription](https://cpl.thalesgroup.com/software-monetization/perpetual-vs-subscription-licenses)

### 8.2 Quando Scegliere Subscription (il Nostro Caso)

**Subscription è MEGLIO quando:**

✅ **Software richiede manutenzione frequente**
- Aggiornamenti normativi (contabilità!)
- Security patches regolari
- Bug fix continui

✅ **Cliente preferisce OPEX vs CAPEX**
- PMI preferiscono costi operativi mensili
- Evitano grosso investimento iniziale
- Budget più prevedibile

✅ **Vendor vuole revenue ricorrente**
- Flusso cassa prevedibile
- Relazione continuativa con cliente
- Lifetime value più alto

✅ **Include hosting e infrastruttura**
- Cliente non ha competenze IT
- Vendor gestisce tutto
- Nessun setup on-premise

**Perpetual è meglio quando:**
- Software stabile, pochi update necessari
- Cliente ha IT interno e vuole controllo
- Uso principalmente offline
- Cliente preferisce possedere outright

**Per ContabilitaAntigravity:** SUBSCRIPTION è la scelta giusta perché:
1. Software contabilità richiede aggiornamenti normativi frequenti
2. Cliente PMI preferisce canone mensile prevedibile
3. Include hosting (cliente non deve gestire server)
4. Supporto continuativo essenziale
5. Revenue ricorrente per vendor

**Fonte:** [How to Choose Between Models](https://licensespring.com/blog/guide/perpetual-license-vs-subscription-license)

## 9. PRICING SUGGERITO: ContabilitaAntigravity

### 9.1 Analisi Costi Vendor

**Costi mensili ricorrenti:**
- Hosting cloud (VPS/cloud): €20-50/mese
- Database hosting: €10-20/mese
- Backup storage: €5-10/mese
- SSL certificate (annuale/12): €5/mese
- Monitoring tools: €10-20/mese
- **Totale infrastruttura:** ~€50-100/mese

**Costi lavoro (stimati):**
- Supporto medio: 2-4 ore/mese → €160-320/mese (@€80/h)
- Manutenzione/bug fix: 1-2 ore/mese → €80-160/mese
- Updates normativi: 2-4 ore/anno → ~€50/mese ammortizzato
- **Totale lavoro:** ~€290-530/mese

**COSTO TOTALE VENDOR:** €340-630/mese

### 9.2 Pricing Proposto

**Modello consigliato:**

```
CANONE BASE: €400-600/mese + IVA

Include:
- Licenza software (utenti illimitati per singola azienda)
- Hosting completo (server, database, SSL, backup)
- Manutenzione ordinaria (bug fix, security updates)
- Aggiornamenti normativi inclusi
- Supporto email/ticket (48h response time)
- 3 ore supporto tecnico/mese incluse
- Backup giornaliero (retention 30gg)

Extra (non incluso):
- Ore supporto oltre 3h/mese: €80/ora
- Nuove funzionalità custom: quotate a progetto
- Formazione on-site: €500/giornata
- Integrazioni sistemi terzi: quorate a progetto

Condizioni:
- Durata minima: 12 mesi
- Pagamento: mensile posticipato
- Rinnovo: automatico mensile dopo anno 1
- Aumento annuale: max 5% (dopo anno 1)
- Cancellazione: 60 giorni preavviso (dopo anno 1)
```

### 9.3 Marginalità

| Scenario | Canone | Costo Vendor | Margine | % Margin |
|----------|--------|--------------|---------|----------|
| **Ottimistico** | €600/mese | €340/mese | €260/mese | 43% |
| **Realistico** | €500/mese | €480/mese | €20/mese | 4% |
| **Pessimistico** | €400/mese | €630/mese | -€230/mese | -58% |

**Nota:** Il margine nel primo anno può essere basso/negativo a causa di setup e supporto iniziale intenso. L'investimento si recupera nei rinnovi successivi dove il supporto richiesto diminuisce.

**Lifetime Value (3 anni):**
- Canone: €500 × 36 mesi = €18,000
- Costo anno 1: ~€630/mese (alto supporto)
- Costo anno 2-3: ~€400/mese (supporto ridotto)
- **Profitto 3 anni:** €18,000 - €19,080 = -€1,080 ❌

**ATTENZIONE:** Con questi numeri, il margine è troppo sottile!

### 9.4 Pricing Corretto (con Margine Sano)

**Prezzo minimo raccomandato:** €650-800/mese

```
TIER UNICO: €750/mese + IVA

Lifetime Value 3 anni:
- Revenue: €750 × 36 = €27,000
- Costi anno 1: €630 × 12 = €7,560
- Costi anno 2-3: €400 × 24 = €9,600
- Totale costi: €17,160
- PROFITTO 3 ANNI: €9,840 ✅
- Margine medio: 36% ✅
```

**Questo pricing è giustificato perché:**
- Software custom (non off-the-shelf)
- Include hosting dedicato
- Aggiornamenti normativi inclusi (valore alto per contabilità)
- Supporto diretto developer
- Nessun limite utenti/transazioni
- Tutto incluso (no costi nascosti)

**Confronto mercato:**
- QuickBooks Online: $38-70/mese (€35-65) → ma è generico, multi-tenant
- Xero: €70/mese → generico
- SAP Business One: non pubblicato, enterprise-level
- **Custom solution PMI italiana:** €500-800/mese è ragionevole

```
Mensile: €750/mese
Annuale prepagato: €8,100/anno (€675/mese, -10%)
```
Pro: Cash flow immediato, commitment cliente
Contro: Cliente PMI potrebbe preferire mensile

### 10.4 Prossimi Step

1. **Valutare pricing finale** in base a:
   - Complessità effettiva del software ContabilitaAntigravity
   - Costi reali supporto (storico se disponibile)
   - Posizionamento desiderato (premium vs accessibile)

2. **Preparare contratto** usando template sopra

3. **Decidere ownership model**:
   - Solo licenza? (vendor mantiene sempre IP)
   - Con exit option? (cliente può acquistare IP)
   - Con escrow? (protezione cliente)

4. **Definire cosa è "extra"**:
   - Chiarire confine manutenzione vs nuove feature
   - Definire cosa rientra nelle 3 ore supporto incluse

5. **Testare pricing** con cliente attuale o prospect

## 12. GLOSSARIO

| Termine | Definizione |
|---------|-------------|
| **ARR** | Annual Recurring Revenue - Revenue ricorrente annuale |
| **Churn** | Tasso di abbandono clienti (cancellazioni subscription) |
| **CPI** | Consumer Price Index - Indice prezzi al consumo |
| **Escrow** | Deposito di source code presso terza parte fidata |
| **Flat-rate** | Prezzo fisso, non dipende da usage |
| **LTV** | Lifetime Value - Valore totale cliente nel tempo |
| **MRR** | Monthly Recurring Revenue - Revenue ricorrente mensile |
| **Per-user** | Pricing basato su numero utenti |
| **SLA** | Service Level Agreement - Livelli di servizio garantiti |
| **Tiered** | Pricing a pacchetti (Small/Medium/Large) |
| **Usage-based** | Pricing basato su utilizzo effettivo |
| **Uptime** | Percentuale tempo di disponibilità sistema |
