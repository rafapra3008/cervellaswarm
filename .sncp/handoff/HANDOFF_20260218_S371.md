## DECISIONI

| Decisione | Perche |
|-----------|--------|
| `role:` field nel frontmatter | Ispirato da CrewAI (best idea). Human-readable label per task routing |
| `team.yaml` formato dichiarativo | 0/5 competitor hanno composizione team dichiarativa. Differenziale unico |
| `permissionMode:` + `maxTurns:` | Campi nativi Claude Code che non usavamo. Plan mode per architect |
| 7 worker specialties | Copertura completa dei ruoli comuni + generic catch-all |
| pyyaml unica dipendenza | Serve per rendering team.yaml. Leggerissimo |
| Template con `{{ variable }}` | Semplice, no Jinja2 dependency, sufficiente per il nostro caso |

---

## LEZIONI APPRESE

1. **team.yaml e un differenziale REALE**: 0 su 5 framework competitor hanno composizione team dichiarativa. E il tipo di feature che crea community buzz
2. **Shared DNA via Markdown batte Python inheritance**: piu leggibile, editabile da non-dev, version-controllabile
3. **Worker specialty con template variables**: `{{ specialty_details }}` evita duplicazione tra 7 speciality diverse
4. **Tautological tests (`assert X or True`)**: la Guardiana ha trovato un test che passava sempre. Lezione: mai `or True` in assert
5. **dist/ e build/ vanno SEMPRE in .gitignore**: mancavano dal root, rischio accidentale commit di binary artifacts
