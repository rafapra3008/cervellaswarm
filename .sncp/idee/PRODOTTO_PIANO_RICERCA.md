## FASE 1: CAPIRE IL TERRENO

### 1. Competitor Analysis (PRIMO)

**Perché primo:** Non possiamo posizionarci se non sappiamo cosa fanno gli altri.

| Competitor | Cosa studiare | Priorità |
|------------|---------------|----------|
| Cursor | Pricing, features, architettura, limiti | ALTA |
| Windsurf | Perché già siamo migliori? Confermare | ALTA |
| Antigravity (Google) | Nuovo, capire cosa offrono | ALTA |
| GitHub Copilot | Il gigante, punti deboli | MEDIA |
| Continue.dev | Open source, come funziona | MEDIA |
| Cody (Sourcegraph) | Enterprise, come si posizionano | BASSA |

**Output atteso:** Documento con mappa competitor, gap di mercato, opportunità.

---

### 2. Anthropic / API (SECONDO)

**Perché secondo:** Dobbiamo sapere cosa possiamo fare legalmente e tecnicamente.

| Cosa | Domanda chiave |
|------|----------------|
| API Pricing | Quanto costa per utente? Margini possibili? |
| Enterprise | Esiste? Cosa include? Context più lungo? |
| Terms of Service | Possiamo rivendere accesso? Come? |
| Partner Program | Esiste? Come entrare? |
| Rate Limits | Quante richieste? Limiti per uso commerciale? |
| Contatto diretto | Vale la pena contattare Anthropic? |

**Output atteso:** Documento con fattibilità tecnica/legale, costi stimati, opzioni partnership.

---

### 3. I Nostri Differenziatori (TERZO)

**Perché terzo:** Dopo aver visto competitor e limiti API, capiamo cosa abbiamo di UNICO.

| Differenziatore | Da validare |
|-----------------|-------------|
| SNCP | Nessuno ce l'ha? Brevettabile? |
| Multi-Agent (16 cervelle) | Altri hanno simile? |
| Metodologia (checkpoint, NORD) | Come comunicarla? |
| "Solo Claude" | È un pro o un contro? |
| Per non-developer | Mercato esiste? |

**Output atteso:** Lista differenziatori validati, come comunicarli.
