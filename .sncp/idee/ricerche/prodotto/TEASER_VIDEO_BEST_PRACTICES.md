# Teaser Video - Best Practices

> **Data:** 9 Gennaio 2026 - Sessione 140
> **Ricerca:** cervella-researcher

---

## 1. DURATA IDEALE

- **Teaser:** 15-60 secondi (sweet spot: 30-45 sec)
- **Developer tools:** < 60 sec massimo
- **Hook iniziale:** primi 3 secondi CRITICI
- **85% video guardati senza audio su mobile**

---

## 2. STRUTTURA VINCENTE

```
HOOK (3 sec)
  -> Domanda/problema relatable o visual shock

PROBLEMA (10-15 sec)
  -> Scenario reale developer

SOLUZIONE (20-30 sec)
  -> Demo workflow in azione

CTA (5 sec)
  -> Azione specifica (trial, sign-up)
```

---

## 3. COSA FUNZIONA (Esempi Dev Tools)

| Tool | Cosa fa bene |
|------|--------------|
| **Snyk** | Integrazione real-time in IDE/pipeline |
| **Vercel** | Workflow "Develop. Preview. Ship." con data viz |
| **FlutterFlow** | UI iPad -> preview iPhone -> deploy App Store |

**Pattern comune:** Storytelling > feature list. Mostra workflow, non singole feature.

---

## 4. ERRORI DA EVITARE

- Troppo testo sullo schermo (diluisce messaggio)
- Information overload (max 1-2 concetti chiave)
- Mancanza CTA chiaro
- Feature dump invece di storia
- Video che richiede audio (92% mobile = mute)
- Inauthenticita' (audience rileva il fake)

---

## 5. BEST PRACTICES TEXT-ONLY (senza voiceover)

- Testo integrato nella scena (non overlay pesante)
- **Max 10 parole per schermata**
- Musica/sound design supporta, non distrae
- Testare A/B con/senza audio
- Ottimizzato per viewing mute

---

## RACCOMANDAZIONE PER CERVELLASWARM

```
Durata: 45-60 secondi
Struttura: Hook -> Problema -> Demo -> CTA
Testo: Minimale, integrato
Audio: Musica sottofondo, NO voiceover
Focus: Coordinazione + Memoria (differenziatori)
```
